Elegir las Herramientas y Tecnologías (No lo hemos hecho) Frontend: HTML, CSS, JavaScript puro o con un framework como React/Vue.js (Para algo dinámico ). Backend (opcional al inicio): Si solo queremos consumir la PokeAPI, podriamos empezar con un proyecto estático y luego agregar un backend (Node, Python, etc.) más adelante. Hosting: El Github: https://sejuaniii.github.io/Pokemon-Origins/ 3. Estructura de Carpetas y Primeros Archivos Crear carpetas como assets (para imágenes, CSS, JavaScript), pages (para manejar múltiples páginas), etc. Preparar un index.html con la estructura base y un enlace a un archivo app.js o similar donde manejaremos la lógica de la Pokedex. Si vamos con React/Vue, generan el proyecto con Create React App, Vite, etc., y definen los componentes. 4. Integrar la PokeAPI (o la fuente de datos que elijamos) Probar la API: Hacer fetch a la PokeAPI (ejemplo: https://pokeapi.co/api/v2/pokemon/1/) y mostrar la información en consola. Renderizar datos: Mostrar al menos nombre, imagen y tipo en el HTML. Diseño básico: Usar CSS para que la presentación sea agradable. 5. Colaboración y Control de Versiones Usar Ramas: Cada uno trabajara en su rama (feature/api, feature/ui, etc.) y luego hacen pull requests a main. Commits pequeños y descriptivos: Facilita el seguimiento de cambios. Revisiones de código: Antes de hacer merge, lo revisamos juntos para mantener la calidad y aprender de los cambios. 6. Próximos Pasos y Roadmap Terminar MVP: Index con una introducción. Pokedex que consume la PokeAPI y muestra un listado de Pokémon. Página de detalle o un modal (opcional) con información adicional. Agregar Funcionalidades Extras: Búsqueda o filtros por tipo. Sección de noticias o guías. Sistema de login (si queremos perfiles de usuario). Optimizar y Escalar: Migrar a un framework si iniciamos con HTML/CSS/JS puro. Añadir un backend para guardar favoritos o comentarios. En Resumen Pasar a la fase de desarrollo del MVP: Definir qué partes tendrá y cómo dividirse las tareas. Elegir herramientas y estructurar el proyecto: Carpetas, archivos y flujo de trabajo con Git. Consumir la PokeAPI: Empezar a traer datos reales y mostrarlos en la página. Mantener la comunicación: Usar issues, pull requests y tal vez un board de tareas (Trello, GitHub Projects, etc.) para organizar el trabajo. ¿Todo claro? ¡A por el “Pokemon-Origins”!
